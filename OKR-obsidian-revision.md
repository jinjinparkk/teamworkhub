# OKR 수정안 (옵시디언·Drive 기준)

아래는 **기존 Jira 버전과 동일한 양식**으로만 문구를 바꾼 것입니다. 스프레드시트에 붙일 때는 따옴표 안 KR만 한 셀에 넣으면 됩니다.

---

## 복사용 (Objective / KR1 / KR2)

```
Objective			팀원 모두가 핵심 업무에 집중할 수 있는 환경을 만든다.
KR1			"이메일이 출근 전 옵시디언·Drive(마크다운)에 자동 정리되는 체계 구축
25% : 다계정·멱등을 포함한 Gmail→Drive/마크다운 동기화 파이프라인 구축 및 개발
50% : 라벨·수집 범위·예외 처리 등 필터링·분류 규칙 개발
75% : 테스트
100% : 라이브 적용"
KR2			"팀원이 출근 즉시 당일 처리 업무와 메일 맥락을 한눈에 확인할 수 있는 옵시디언 기반 데일리·할 일 체계 구축
25% : 데일리·가시성 요구사항 정의 및 볼트/템플릿(UI) 설계
50% : 데일리 요약·per-mail 노트 등 마크다운 생성 기능 개발
75% : Drive/볼트 동기 경로 정리 및 태그·담당자·Dataview 연계 메타 규칙 반영
90% : 테스트
100% : 라이브 적용"
```

## 메일 초안 (OKR 수정·개발 범위 확대 안내)

**제목:** [OKR 수정] 이메일·당일 업무 가시화 목표 — 도구 정합 및 개발 범위 조정

안녕하세요,

분기 OKR 중 “이메일 자동 정리” 및 “출근 직후 당일 업무 확인” 항목과 관련해, 문구 수정을 제안드립니다.

**1. 도구 정합**  
팀은 Jira가 아닌 **옵시디언**을 기준으로 할 일·데일리를 운영하고 있어, 기존 KR의 Jira 연동 표현은 실제 업무와 맞지 않습니다. 목표 의도는 유지하되, **Gmail → Drive/마크다운 → 옵시디언 볼트** 흐름에 맞게 KR을 바꾸었습니다.

**2. 기존 OKR 대비 개발 범위를 늘린 부분**  
초안은 “Jira에 자동 정리”처럼 **단일 연동**에 가깝게 읽힐 여지가 있었는데, 실제로는 팀에서 쓰기 위해 **그 이상의 개발·운영 범위**가 필요합니다. 예를 들어 **다계정 Gmail**, **중복 없이 안전하게 쌓는 멱등 처리**, **첨부·본문·데일리 요약까지 포함한 파이프라인**, **클라우드에서 스케줄 실행**, **옵시디언 검색·태그와 맞는 메타 규칙** 등이 포함됩니다. 즉, 이름만 바꾼 것이 아니라 **실사용을 전제로 한 구현·검증·라이브까지의 개발 물량을 OKR에 명시적으로 반영**한 형태입니다.

첨부한 수정안(Objective·KR1·KR2) 기준으로 검토 부탁드리며, 피드백 주시면 반영하겠습니다.
